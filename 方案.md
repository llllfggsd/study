下面可以整理成一版比较完整的 **学习题库网站方案** ，核心目标是：

**用户登录后，创建自己的学习分类，导入 XLSX（Excel 表格文件）题库，然后可以顺序练习、随机练习、查看错题集。**

---

# 学习题库网站方案

## 一、网站定位

建设一个面向个人学习的题库练习网站。

用户可以按照不同学习方向创建模块，例如：

```text
机器学习
Python
历史
英语
数学
自定义分类
```

每个模块下可以导入选择题，题目来源为 XLSX 文件。

系统支持：

```text
1、用户登录
2、用户数据隔离
3、学习分类管理
4、题目导入
5、顺序做题
6、随机做题
7、错题集
8、答题记录
```

---

### 4、题目导入页

导入文件格式固定为 XLSX。

表头必须是：

```text
问题、A、B、C、D、答案、解析
```

示例：

| 问题                            | A          | B          | C      | D          | 答案 | 解析                        |
| ------------------------------- | ---------- | ---------- | ------ | ---------- | ---- | --------------------------- |
| Python 中定义变量的正确方式是？ | int a = 10 | let a = 10 | a = 10 | var a = 10 | C    | Python 变量不需要声明类型。 |

导入规则：

```text
1、第一行必须是标题
2、问题、A、B、C、D、答案不能为空
3、答案只能是 A、B、C、D
4、解析可以为空
5、导入题目自动绑定当前用户
6、导入题目自动绑定当前分类
```

## 三、做题功能设计

### 1、顺序做题

顺序做题按照题目创建顺序展示。

```sql
SELECT *
FROM questions
WHERE user_id = 当前用户ID
  AND category_id = 当前分类ID
ORDER BY id ASC;
```

适合系统学习。

---

### 2、随机做题

随机做题每次打乱题目顺序。

```sql
SELECT *
FROM questions
WHERE user_id = 当前用户ID
  AND category_id = 当前分类ID
ORDER BY RANDOM();
```

适合复习巩固。

---

### 3、答题页面

页面示例：

```text
Python 基础题库

第 1 / 80 题

问题：
Python 中定义变量的正确方式是？

A. int a = 10
B. let a = 10
C. a = 10
D. var a = 10

[提交答案]
```

提交后显示：

```text
回答错误

你的答案：B
正确答案：C

解析：
Python 定义变量时不需要声明类型，直接赋值即可。

[下一题]
```

## 四、错题集设计

错题集不需要单独导入，也不需要手动维护。

只要用户答错，系统就记录到答题记录里。

查询错题时，根据答题记录自动生成。

```sql
SELECT q.*
FROM questions q
JOIN question_records r ON q.id = r.question_id
WHERE r.user_id = 当前用户ID
  AND r.is_correct = 0
GROUP BY q.id;
```

错题集页面：

```text
Python 错题集

错题数量：9

[开始错题练习]
[清空错题记录]
```

## 五、数据库设计

使用 SQLite（轻量级本地数据库）存储数据。

### 1、用户表

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

---

### 2、学习分类表

```sql
CREATE TABLE categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    name TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

---

### 3、题目表

```sql
CREATE TABLE questions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    category_id INTEGER NOT NULL,

    question TEXT NOT NULL,
    option_a TEXT NOT NULL,
    option_b TEXT NOT NULL,
    option_c TEXT NOT NULL,
    option_d TEXT NOT NULL,
    answer TEXT NOT NULL,
    explanation TEXT,

    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (category_id) REFERENCES categories(id)
);
```

---

### 4、答题记录表

```sql
CREATE TABLE question_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    question_id INTEGER NOT NULL,

    user_answer TEXT NOT NULL,
    is_correct INTEGER NOT NULL,

    answered_at DATETIME DEFAULT CURRENT_TIMESTAMP,

    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (question_id) REFERENCES questions(id)
);
```

## 六、核心数据关系

```text
用户 users
  ↓
学习分类 categories
  ↓
题目 questions
  ↓
答题记录 question_records
  ↓
错题集
```

最重要的是：

```text
categories 表有 user_id
questions 表有 user_id
question_records 表有 user_id
```

这样才能保证不同用户的数据完全分开。

## 七、推荐技术结构

可以采用简单结构：

```text
前端页面
↓
后端接口
↓
SQLite 数据库
```

推荐技术：

```text
前端：Vue（前端页面框架）或 React（前端页面框架）
后端：FastAPI（Python 后端接口框架）
数据库：SQLite（轻量级本地数据库）
题目导入：openpyxl（Python 读取 Excel 的库）
登录验证：Session（登录状态）或 JWT（登录令牌）
```

如果只是自己或小范围使用，建议：

```text
Vue + FastAPI + SQLite
```

## 八、后端接口设计

### 用户相关

```text
POST /register
用户注册

POST /login
用户登录

POST /logout
退出登录

GET /me
获取当前登录用户信息
```

---

### 分类相关

```text
GET /categories
获取当前用户的分类列表

POST /categories
新建分类

DELETE /categories/{category_id}
删除分类
```

---

### 题目相关

```text
POST /categories/{category_id}/import
导入 XLSX 题目

GET /categories/{category_id}/questions
获取题目列表

DELETE /questions/{question_id}
删除题目
```

---

### 练习相关

```text
GET /categories/{category_id}/practice?mode=order
顺序练习

GET /categories/{category_id}/practice?mode=random
随机练习

POST /questions/{question_id}/answer
提交答案
```

---

### 错题相关

```text
GET /categories/{category_id}/wrong-questions
获取错题集

POST /questions/{question_id}/remove-wrong
从错题集中移除

DELETE /categories/{category_id}/wrong-records
清空当前分类错题记录
```

## 十、核心重点

这个网站的核心不是页面复杂，而是这条流程要完整：

```text
登录用户
→ 创建学习分类
→ 导入 XLSX 题目
→ 选择顺序 / 随机练习
→ 提交答案
→ 记录答题结果
→ 自动生成错题集
→ 反复练习错题
```

只要这条流程打通，这个学习网站就可以正常使用。
